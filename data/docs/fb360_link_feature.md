# How to link feedback to a feature request in FB360

## Overview
Linking feedback to feature requests is one of the most important actions in FB360. When multiple feedback entries from different clients are linked to the same feature request, it builds a powerful signal about demand. This signal directly influences how engineering and product teams prioritise their roadmap.

## Why linking matters
- A feature requested by 10 clients carries far more weight than one requested by 1 client
- The total revenue opportunity across all linked feedback is aggregated on the feature record
- Engineering leaders and release managers see the aggregated signal when prioritising
- Without linking, individual feedback entries are isolated data points with limited influence

## Step-by-step instructions

### Step 1 — Open the feedback record
Navigate to the feedback entry you want to link. Use the search bar or filter by Feedback ID, client name, or feedback type.

### Step 2 — Scroll to the "Linked feature" section
On the feedback detail page, scroll to the **Linked feature** panel.

### Step 3 — Search for an existing feature request
Click **Link feature** and type keywords in the search dialog. Search by feature title or category. If a matching feature exists:
- Review the description to confirm it matches the client's feedback
- Click **Link** to associate the feedback with the feature

### Step 4 — Create a new feature request if needed
If no matching feature is found, click **Create new feature**. Fill in:
- **Title** — a product-focused description of the capability
- **Category** — AI/ML, Security, Performance, Integration, Analytics, Compliance, Developer Experience
- **Description** — what the feature should do, written from a user perspective
- **Complexity** — your estimate: Low, Medium, High, Very High

### Step 5 — Confirm the revenue opportunity
After linking, confirm that the revenue opportunity figure on the feedback record is accurate. This figure is rolled up to the feature's total revenue opportunity automatically.

### Step 6 — Add a linking comment
Add a comment explaining why this feedback is linked to the selected feature. Include:
- How the client described their need
- Whether they mentioned specific use cases
- How urgent the need is for this client

## How the aggregated signal works
Once linked, the feature record shows:
- **Total feedback count** — how many feedback entries are linked
- **Total revenue opportunity** — the sum across all linked feedback records
- **Client list** — which clients have requested this feature
- **Sentiment breakdown** — how many feedback entries are positive, neutral, or negative

Release managers use this information when deciding what to build next.

## Tips
- Always search before creating a new feature — duplicate feature requests split the signal.
- If a client mentions a feature in passing (not a formal request), still log it in FB360 and link it. Volume of mentions matters even without strong revenue figures.
- Review the existing feature list monthly — features you thought were unique may already exist.
